共通チェック (COM)
COM-01 デフォルトタイトル残存 WARN

列/行仕様のタイトルにデフォルト値（"Column/Row Title" / "列/行タイトル"）が残っています。

fusion_place がフォームを生成したときの初期値がそのまま残っている状態です。タイトルは画面上の列・行ヘッダに表示されるため、そのままでは利用者に伝わりません。適切な名称に変更してください。

COM-02 起点メンバと展開方法の組み合わせ WARN / INFO

ループの起点メンバ（peg）と展開方法（expansion-method）の組み合わせが不正または非推奨です。

有効な組み合わせ

  • ALL → NONE_EXPANSION のみ有効
  • ROOTS → NONE_EXPANSION のみ有効
  • DESCENDENT_BF → #FY・#PERIOD ディメンション以外では非推奨

INFO として通知されるケース

  • 起点メンバ ROOTS の使用（めったに使用しない設定）
  • 展開方法 DESCENDENT_BF の使用（めったに使用しない設定）
COM-03 元帳マスク設定 INFO

フォームルートの local-mask に値が設定されています。

元帳マスクはデータの表示範囲を絞り込む設定です。意図的に設定している場合は問題ありませんが、開発中のフォームに意図せず残っていないか確認してください。

COM-04 トリガー設定 WARN

フォームにトリガーが設定されています。

トリガーはフォーム操作に連動して自動処理（計算実行・インポートなど）を起動します。意図しないタイミングで処理が走らないよう、設定内容を確認してください。

COM-05 エクスポートの行ループ項目出力設定 WARN

エクスポートが有効なのに、行ループ項目の出力が false になっています。

エクスポートフォームで行ループを使用している場合、export-row-loop-items を true にしないとループ内の行データがエクスポートされません。意図して false にしている場合を除き、true に変更してください。

COM-06a 記述ルール: メソッドはすべて小文字 WARN

テキスト式中のメソッド呼び出し（.methodName 形式）に大文字が含まれています。

fusion_place の式言語ではメソッド名は大文字・小文字を区別しますが、規約としてすべて小文字で記述します。

✗ !ACCOUNT.Label ✓ !ACCOUNT.label

自動修正機能（🔧）で一括修正できます。

COM-06b 記述ルール: ラベルはすべて大文字 WARN

テキスト式中のラベル参照（!label 形式）に小文字が含まれています。

メンバラベルはすべて大文字で記述するのが規約です。

✗ !account.label ✓ !ACCOUNT.label

自動修正機能（🔧）で一括修正できます。

COM-06c 記述ルール: #LEAFの値は "TRUE"/"FALSE" ERROR

#LEAF の比較値が "TRUE" または "FALSE"（ダブルクォート付き大文字）ではありません。

#LEAF は文字列型のシステムメンバです。必ずダブルクォートで囲み、大文字で記述してください。

✗ #LEAF = TRUE // 引用符なし ✗ #LEAF = "true" // 小文字 ✗ #LEAF = "True" // 混在 ✓ #LEAF = "TRUE" ✓ #LEAF = "FALSE"
COM-06d 記述ルール: 予約語はすべて小文字 WARN

テキスト式中の予約語に大文字が含まれています。

以下の予約語はすべて小文字で記述します。

ifthenelseendif andornot dimensionsledgerseditions truefalse

ただし #LEAF 等との比較に使う文字列定数 "TRUE"/"FALSE" はダブルクォート内なので対象外です。

✗ IF !ACCOUNT.isleaf = "TRUE" THEN 1 ELSE 0 ENDIF ✓ if !ACCOUNT.isleaf = "TRUE" then 1 else 0 endif

自動修正機能（🔧）で一括修正できます。

COM-06e 記述ルール: 関数はすべて小文字 WARN

テキスト式中の関数名（@function 形式）に大文字が含まれています。

@CUR・@POV・@RKEY は疑似メンバラベル（COM-06f 参照）のため除外されます。それ以外の関数はすべて小文字で記述します。

✗ @Sum(!ACCOUNT, dimensions) ✓ @sum(!ACCOUNT, dimensions)

自動修正機能（🔧）で一括修正できます。

COM-06f 記述ルール: 疑似メンバラベルはすべて大文字 WARN

疑似メンバラベル（@CUR・@POV・@RKEY）に小文字が含まれています。

これらは関数ではなくメンバラベルとして扱われるため、大文字で記述します。

@CUR@POV@RKEY
✗ !ACCOUNT!@cur.label ✓ !ACCOUNT!@CUR.label

自動修正機能（🔧）で一括修正できます。

COM-07 フォームラベルのハイフン WARN

フォームの label 属性にハイフン（-）が含まれています。

ハイフンはアンダースコア（_）に置き換えることを推奨します。ハイフンを含むラベルはスクリプトや他フォームからの参照時に誤動作する場合があります。

✗ label="form-name-001" ✓ label="form_name_001"
COM-08 セルを保護 WARN

帳票書式の「セルを保護」が有効になっています。

セルを保護するとユーザーがフォーム上でセルを直接編集できなくなります。インポートフォームや入力フォームで意図せず有効になっていないか確認してください。

COM-09 ループインデント幅設定が無効 WARN

行ループの row-title-indent が帳票書式の同設定より小さく、インデントが効きません。

帳票書式の row-title-indent がループの設定値の上限として機能します。ループに設定したインデント幅が帳票書式の値以下の場合、視覚的なインデントは現れません。ループの値を帳票書式の値より大きくするか、帳票書式側を小さくしてください。

COM-10 列/行仕様のラベル・名称設定 INFO

列仕様または行仕様の label 属性と name 要素がともに未設定です。

スペーサー行や計算用の非表示行として意図的に空にしている場合は問題ありません。そうでない場合はラベルまたは名称を設定してください。仕様が1件のみの場合はチェックをスキップします。

COM-11 数値のみのラベル ERROR

列/行仕様の label が数値のみで構成されています。

fusion_place の式ではオフセット参照（例: 1 行上を参照）と同じ書式になるため、数値のみのラベルは数式から正しく参照できません。ラベルには必ず文字を含めてください。

✗ label="1" ✓ label="row1" ✓ label="col_01"
参照用フォームのチェック (REF)
REF-01 タイトル未設定 WARN / INFO

列/行仕様の title が未設定です。

参照用フォームでは列・行のタイトルが画面表示に使われます。非表示（suppressed="true"）の仕様の場合は INFO として通知します。

REF-02 ループ項目選択行の表示設定 INFO

行ループの item-addition-prohibited が true に設定されていません。

参照用フォームではユーザーがループ項目を追加できないようにするのが一般的です（item-addition-prohibited="true"）。意図的に許可している場合は問題ありません。

REF-03 帳票書式 ループヘッダ表示設定 INFO

帳票書式の row-wise-loop-layout が LOOP_HEADERS（デフォルト）になっています。

参照用フォームでは FIRST_DETAILS が一般的な設定です。LOOP_HEADERS のままだとループヘッダ行のみが表示される場合があります。意図的に設定している場合は問題ありません。

REF-04 ドリルダウン条件未設定 WARN

ドリルダウンが有効（enabled="true"）なのに condition が空です。

ドリルダウンの実行条件が設定されていないと、意図しないタイミングでドリルダウンが発動する可能性があります。適切な条件式を設定してください。

インポートフォームのチェック (IMP)
IMP-01 インポート仕様が無効 ERROR

import-spec が存在しないか、enabled が true ではありません。

インポートパターンとして選択したフォームに import-spec が存在しない、または無効になっています。インポートフォームとして機能させるには import-spec > enabled を true に設定してください。

IMP-02 符号表示タイプ WARN

インポート仕様の sign-type が ACCOUNT（勘定科目属性に従う）になっていません。

符号表示タイプは通常 ACCOUNT に設定します。異なる設定の場合、インポート時に符号が反転するなどの問題が発生する可能性があります。

IMP-03 ヘッダ読み飛ばし行数 INFO

インポート仕様の num-of-header-lines の値を通知します。

インポートするCSVファイルのヘッダ行数と一致しているか確認してください。

エクスポートフォームのチェック (EXP)
EXP-01 エクスポート仕様が無効 ERROR

export-spec が存在しないか、enabled が true ではありません。

エクスポート・パイプラインパターンとして選択したフォームに export-spec が存在しない、または無効になっています。export-spec > enabled を true に設定してください。

EXP-02 列ヘッダ出力抑制 WARN

suppress-column-headers が true になっており、列ヘッダがエクスポートされません。

後続処理でヘッダが必要な場合はこの設定を解除してください。パイプライン用途など意図的に抑制している場合は問題ありません。

EXP-03 行ヘッダ出力抑制 WARN

suppress-row-headers が true になっており、行ヘッダがエクスポートされません。

後続処理でヘッダが必要な場合はこの設定を解除してください。意図的に抑制している場合は問題ありません。

パイプライン出力フォームのチェック (PIP)
PIP-01 行ループ表示仕様 .label ERROR

行ループの title が未設定、または .label 形式になっていません。

パイプライン出力フォームでは、インポート先フォームがループのメンバをディメンションとして識別できるよう、ループの title に .label 形式を指定する必要があります。

// ループtitleの正しい書き方 ✓ en;"ACCOUNT!@CUR.label" ✗ en;"ACCOUNT!@CUR.description" // .label でない ✗ （未設定）

インポート先でもディメンションとして扱う場合、.label 以外で出力するとインポート時にエラーになります。
