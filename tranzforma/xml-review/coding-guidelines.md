# fusionplace フォーム コーディング規約

> このドキュメントは、fusionplace のフォーム定義（XML）を作成・編集する際のルールをまとめたものです。
> xml-review ツールの自動チェック（COM-06 系）に対応しています。

---

## 1. 式の書き方ルール

フォームの「タイトル式」「数式」「メンバー条件」などに記述する式には、以下のルールがあります。

---

### 【ルール 1】メソッドはすべて小文字で書く（COM-06a）

ディメンションメンバーに対して `.label` や `.value` のように「.（ドット）」に続けて書く部分を**メソッド**といいます。
メソッドはすべて**小文字**で書いてください。

| ❌ NG | ✅ OK |
|---|---|
| `!ACCOUNT.Label` | `!ACCOUNT.label` |
| `!ACCOUNT.Value` | `!ACCOUNT.value` |
| `@CUR.ITEM` | `@CUR.item` |

---

### 【ルール 2】ラベルはすべて大文字で書く（COM-06b）

「`!`（エクスクラメーション）」の後に書くメンバーラベルは**すべて大文字**で書いてください。

| ❌ NG | ✅ OK |
|---|---|
| `!#account` | `!#ACCOUNT` |
| `!#fy` | `!#FY` |
| `!cat` | `!CAT` |

> **例外**：`@CUR`、`@POV`、`@RKEY` で始まるものはこのルールの対象外です（ルール 6 を参照）。

---

### 【ルール 3】#LEAF の値は `"TRUE"` または `"FALSE"` と書く（COM-06c）

`#LEAF` を条件として使う場合、値は必ず**ダブルクォーテーション付き・大文字**で書いてください。

| ❌ NG | 理由 |
|---|---|
| `#LEAF = TRUE` | クォーテーションがない |
| `#LEAF = "true"` | 小文字はエラー |
| `#LEAF = "True"` | 小文字混じりはエラー |

| ✅ OK |
|---|
| `#LEAF = "TRUE"` |
| `#LEAF = "FALSE"` |

---

### 【ルール 4】予約語はすべて小文字で書く（COM-06d）

以下の**予約語**（プログラムが特別な意味として認識する単語）は、すべて**小文字**で書いてください。

| 予約語 | 用途 |
|---|---|
| `if` / `then` / `else` / `endif` | 条件分岐 |
| `and` / `or` / `not` | 論理演算 |
| `true` / `false` | 真偽値（文字列定数 `"TRUE"` とは別） |
| `dimensions` / `ledgers` / `editions` | コレクション参照 |

| ❌ NG | ✅ OK |
|---|---|
| `IF ~ THEN ~` | `if ~ then ~` |
| `AND` | `and` |
| `OR` | `or` |

> **注意**：`#LEAF = "TRUE"` の `"TRUE"` は文字列定数なので大文字です（ルール 3 を参照）。

---

### 【ルール 5】関数はすべて小文字で書く（COM-06e）

「`@`（アットマーク）」に続けて書く**関数**は、すべて**小文字**で書いてください。

| ❌ NG | ✅ OK |
|---|---|
| `@TEXT` | `@text(...)` |
| `@ROUND(...)` | `@round(...)` |
| `@FIELD_KEY(n)` | `@field_key(n)` |

> **例外**：`@CUR`、`@POV`、`@RKEY` は通常の関数とは区別して定義されているため、このルールの対象外です（次のルール 6 を参照）。

---

### 【ルール 6】@関数（疑似メンバラベル）はすべて大文字で書く（COM-06f）

`@CUR`、`@POV`、`@RKEY` は、@関数と呼ばれ、関数の一種ですが、実体は、「現在のメンバー」などを指す特殊なラベル（**疑似メンバーラベル**）に近い性質を持ちます。
公式マニュアルの記載形式に合わせ、これらはすべて**大文字**で書くことにします。

| ❌ NG | ✅ OK |
|---|---|
| `@cur.label` | `@CUR.label` |
| `@pov` | `@POV` |
| `@rkey` | `@RKEY` |

> ドット以降のメソッド部分（`.label` など）は小文字のままでOKです（ルール 1 との組み合わせ）。

---

## 2. フォーム設計ルール

式の書き方以外に、フォームの構成に関するルールです。

---

### 【ルール 7】フォームラベルにハイフンを使わない（COM-07）

フォームのラベル（ID）には **`-`（ハイフン）** を使わず、**`_`（アンダースコア）** を使ってください。

ハイフンはソート順のずれや、スクリプトでの誤動作の原因になることがあります。

| ❌ NG | ✅ OK |
|---|---|
| `SALES-REPORT` | `SALES_REPORT` |
| `budget-form-2024` | `BUDGET_FORM_2024` |

---

### 【ルール 8】列/行仕様のラベルに数字だけを使わない（COM-11）

列や行の仕様に付けるラベルを、数字だけにしないでください。
数字だけのラベルは「オフセット（何列目・何行目）」という意味に解釈されてしまい、数式で正しく参照できなくなります（ver.15以降）。

| ❌ NG | ✅ OK |
|---|---|
| `1` | `ROW01` |
| `100` | `PREV_SALES` |

---

### 【ルール 9】デフォルトタイトルを残さない（COM-01）

フォームを作成したとき、列・行仕様のタイトルに自動的に入る `Column/Row Title`（または `列/行タイトル`）という初期値は、
必ず適切な内容に書き換えてください。フォーム設計画面を見たときにそれぞれの行・列仕様が何を表すのかがすぐわかるようにしておくことが大切です。

もちろん調査用のフォーム等や、移行用の一時的なフォームについてはこの限りではありません

## 3. ルール対応表（チェックコード一覧）

| コード | 重要度 | 内容 | 自動修正 |
|---|---|---|---|
| COM-06a | WARN | メソッドは小文字 | ✅ あり |
| COM-06b | WARN | ラベルは大文字 | ✅ あり |
| COM-06c | ERROR | #LEAF の値は "TRUE"/"FALSE" | ❌ なし |
| COM-06d | WARN | 予約語は小文字 | ✅ あり |
| COM-06e | WARN | 関数は小文字 | ✅ あり |
| COM-06f | WARN | 疑似メンバーラベルは大文字 | ✅ あり |
| COM-07 | WARN | フォームラベルにハイフン不可 | ❌ なし |
| COM-11 | ERROR | ラベルに数字のみ不可 | ❌ なし |
| COM-01 | WARN | デフォルトタイトル残存 | ❌ なし |

> **重要度の目安**
> - **ERROR**：そのまま使うと誤動作・エラーの原因になります。必ず修正してください。
> - **WARN**：推奨されない書き方です。修正することを強くお勧めします。
> - **INFO**：参考情報です。意図的な設定であれば問題ありません。

---

## 4. 自動修正について

xml-review ツールには、**WARN レベルの式ルール（COM-06a/b/d/e/f）** を自動で修正する機能があります。
チェック実行後に「自動修正」ボタンが有効になった場合は、ワンクリックで修正済みの XML をコピーできます。

ただし、以下は自動修正されません。手動での確認・修正が必要です。

- `#LEAF` の値の誤り（COM-06c）
- フォームラベルのハイフン（COM-07）
- 数字のみのラベル（COM-11）
- デフォルトタイトルの残存（COM-01）
